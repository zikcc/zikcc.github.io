📌 字符设备驱动开发基础

1. 驱动类型与作用

三大驱动类型：

类型 访问方式 典型设备 核心操作
字符设备 字节流（顺序） LED、按键、串口 open/read/write/ioctl
块设备 固定数据块 硬盘、SD卡 缓冲区管理、IO调度
网络设备 数据包 网卡、WiFi模块 ndo_start_xmit（发包）

驱动核心作用：

硬件抽象层：驱动程序将硬件寄存器操作封装为标准文件接口（如 /dev/led），实现 “应用与硬件解耦”。
通俗比喻：驱动像“翻译官”，把应用程序的read/write命令翻译成硬件能理解的信号（如GPIO电平控制）。

2. 字符设备驱动开发流程

完整代码模板（带逐行解析）：

#include <linux/module.h>
#include <linux/fs.h>        // 文件操作结构体file_operations
#include <linux/cdev.h>      // 字符设备管理cdev

#define DEVICE_NAME "mydev"  // 设备节点名称

static int major;            // 主设备号
static struct cdev my_cdev;  // 字符设备对象

// 1. 实现设备操作函数（以open为例）
static int my_open(struct inode *inode, struct file *filp) {
    printk(KERN_INFO "Device opened\n");
    return 0;
}

// 2. 定义file_operations结构体（驱动核心！）
static struct file_operations fops = {
    .owner = THIS_MODULE,    // 防止模块被卸载时设备正在使用
    .open = my_open,         // 映射open系统调用到my_open函数
    .read = my_read,         // 需自定义read函数
    .write = my_write,       // 需自定义write函数
};

// 3. 模块初始化（驱动入口）
static int __init my_init(void) {
    // 动态分配设备号（避免冲突）
    dev_t devno;
    if (alloc_chrdev_region(&devno, 0, 1, DEVICE_NAME) < 0) {
        return -1;
    }
    major = MAJOR(devno);   // 提取主设备号

    // 关联cdev与fops
    cdev_init(&my_cdev, &fops);
    if (cdev_add(&my_cdev, devno, 1) < 0) {
        unregister_chrdev_region(devno, 1); // 失败时释放设备号
        return -1;
    }
    return 0;
}

// 4. 模块卸载（驱动出口）
static void __exit my_exit(void) {
    cdev_del(&my_cdev);                  // 删除cdev
    unregister_chrdev_region(MKDEV(major, 0), 1); // 释放设备号
}

module_init(my_init);
module_exit(my_exit);
MODULE_LICENSE("GPL");  // 必须声明GPL协议

技术解析：

  • file_operations：驱动核心结构体，将系统调用映射到硬件操作函数（如read()→my_read()）。
  • 设备号管理：
    • 主设备号：标识驱动类型（如250代表LED驱动）
    • 次设备号：区分同一驱动的多个实例（如LED1、LED2）
    • alloc_chrdev_region()动态分配避免冲突（优于静态分配）。
  • cdev生命周期：cdev_init()→cdev_add()注册设备 →cdev_del()销毁。

3. 面试题与实战

题目1：为什么字符设备驱动必须实现 file_operations？
答案：

内核通过 file_operations 将系统调用（如 open()）映射到驱动的硬件操作函数，实现 硬件抽象层。没有它，应用程序无法通过标准接口（如 /dev/led）控制硬件。

题目2：以下代码有何问题？

static int my_open(struct inode *inode, struct file *filp) {
    printk("Device opened\n");  // 未指定日志级别
    return 0;
}

答案：

错误：printk未指定日志级别（如KERN_INFO），可能导致日志被过滤。
修正：printk(KERN_INFO "Device opened\n");。

4. 动手实验：虚拟字符设备驱动

目标：实现一个虚拟设备，read()时返回字符串 "Hello Driver!"。
步骤：

  1. 补充 my_read 函数：
static ssize_t my_read(struct file *filp, char __user *buf, size_t len, loff_t *off) {
    const char *msg = "Hello Driver!";
    size_t msg_len = strlen(msg);
    // 拷贝数据到用户空间（防止内存越界）
    if (copy_to_user(buf, msg, msg_len) != 0) {
        return -EFAULT;  // 拷贝失败返回错误码
    }
    return msg_len;      // 返回实际读取长度
}
```  
2. 编译加载驱动：  
```bash
make                  # 使用内核Makefile编译
sudo insmod mydev.ko  # 加载驱动
dmesg | tail          # 查看内核日志，确认设备号
sudo mknod /dev/mydev c 250 0  # 创建设备节点（250需替换为实际主设备号）
```  
3. 测试驱动：  
```bash
cat /dev/mydev  # 应输出 "Hello Driver!"
