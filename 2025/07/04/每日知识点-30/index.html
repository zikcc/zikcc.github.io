🔧 一、硬件与处理器架构

题目1 - 中断管理
类型：简答题
题目：解释中断嵌套的概念，并说明在ARM Cortex-M中如何配置中断优先级以实现嵌套。
答案：

  • 中断嵌套：高优先级中断可打断正在执行的低优先级中断处理程序（ISR），形成嵌套。
  • 配置方法：
    • 设置NVIC优先级分组（如4位抢占优先级+0位子优先级）。
    • 高抢占优先级中断可抢占低优先级中断。

技术解析：

// 配置UART中断优先级为2（高），按键中断优先级为1（低）
HAL_NVIC_SetPriority(UART_IRQn, 2, 0); 
HAL_NVIC_SetPriority(EXTI0_IRQn, 1, 0);
```  
**应用场景**：  
实时数据采集（UART收数据中断抢占按键中断）。  

---

### ⚙️ 二、RTOS核心机制  
**题目2 - 任务同步**  
**类型**：代码分析题  
**题目**：以下FreeRTOS代码有何问题？如何修正？  
```c
void TaskA() {
    xSemaphoreTake(mutex, portMAX_DELAY);
    // 操作共享资源
    xSemaphoreGive(mutex);  // 忘记释放信号量
}
```  
**答案**：  
- **错误**：任务在异常分支中可能未释放互斥锁，导致死锁。  
- **修正**：  
  ```c
  void TaskA() {
      if(xSemaphoreTake(mutex, pdMS_TO_TICKS(100)) == pdTRUE) {
          // 操作资源
          xSemaphoreGive(mutex);  // 确保释放
      }
  }
  ```  
**技术解析**：  
- **关键点**：互斥锁必须成对使用，且需考虑超时机制。  
- **应用场景**：SPI总线共享访问（多任务竞争时避免总线冲突）。

### 💻 三、C/C++编程  
**题目3 - 指针与内存**  
**类型**：选择题  
**题目**：以下代码输出什么？  
```c
int a = 10;
int *p = &a;
int **pp = &p;
printf("%d", **pp);
```  
A. 10  B. a的地址  C. p的地址  D. 编译错误  
**答案**：A  

**通俗解释**：  
`pp`是二级指针，指向指针`p`，`p`指向`a`，因此`**pp`解引用两次得到`a`的值。  

**技术解析**：  
```c
int ***ppp = &pp;  // 三级指针（实际开发慎用）
```  
**应用场景**：  
动态多维数组（如图像处理中的矩阵操作）。

### 📡 四、通信协议  
**题目4 - SPI与I2C对比**  
**类型**：简答题  
**题目**：列举SPI与I2C的3点核心区别，并说明各自适用场景。  
**答案**：  
| **特性**       | SPI                          | I2C                  |  
|----------------|------------------------------|----------------------|  
| **线路数量**   | 4线（SCK/MISO/MOSI/SS）      | 2线（SDA/SCL）       |  
| **速度**       | 快（可达50MHz）              | 慢（通常≤400KHz）    |  
| **拓扑结构**   | 一主多从（需片选信号）       | 多主多从（地址寻址） |  

**典型应用**：  
- **SPI**：高速ADC数据采集（如传感器阵列）。  
- **I2C**：低速设备控制（如EEPROM、温度传感器）。  

---

### 🛠️ 五、系统设计与优化  
**题目5 - 低功耗设计**  
**类型**：设计题  
**题目**：设计一个电池供电的物联网设备，待机功耗需≤1μA。列举3种硬件+3种软件优化策略。  
**答案**：  
- **硬件策略**：  
  1. 选用Cortex-M0+内核（超低静态功耗）。  
  2. 关闭未用外设的时钟（如ADC、UART）。  
  3. 使用LDO替代DC-DC（无开关损耗）。  
- **软件策略**：  
  1. 启用RTOS的Tickless模式（空闲时停时钟）。  
  2. 中断唤醒替代轮询（CPU休眠时间最大化）。  
  3. 动态降频（任务少时降低CPU主频）。  

**技术解析**：  
```c
// FreeRTOS启用Tickless模式
configUSE_TICKLESS_IDLE = 1;
```

### 💎 六、综合实战题  
**题目6 - 系统启动流程**  
**类型**：简答题  
**题目**：画出嵌入式Linux启动流程图（从Bootloader到用户程序），并说明各阶段任务。  
**答案**：  
```mermaid
graph LR
A[Bootloader] --> B[加载内核]
B --> C[挂载根文件系统]
C --> D[启动init进程]
D --> E[执行用户程序]
```  
- **Bootloader**：初始化DDR、时钟，加载内核（如U-Boot）。  
- **内核**：驱动硬件，建立进程管理。  
- **根文件系统**：提供/dev、/bin等目录支持应用运行。
