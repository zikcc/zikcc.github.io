📚 一、RTOS核心概念

  1. 实时性分类

    • 硬实时：超时=系统失效（如刹车控制、心脏起搏器），响应时间严格可控。
    • 软实时：超时可容忍（如视频播放），需尽快响应但无致命后果。

  2. RTOS vs 通用OS

    特性 RTOS (FreeRTOS) Linux
    响应时间 微秒级（硬实时保证） 毫秒级（非实时）
    内核大小 KB级（如FreeRTOS 10KB） MB级
    调度策略 固定优先级抢占 CFS公平调度
    适用场景 工业控制、传感器采集 图形界面、服务器

  3. 内核类型

    • 可剥夺型内核：高优先级任务可抢占低优先级任务（FreeRTOS默认）。
    • 不可剥夺型内核：任务主动释放CPU才切换，易引发优先级反转。

⚙️ 二、任务管理（FreeRTOS核心）

  1. 任务四态模型

    graph LR
  A[就绪 Ready] --> B[运行 Running]
  B --> C[阻塞 Blocked]
  C --> A
  B --> D[挂起 Suspended]
  D --> A
    • 运行→阻塞：调用 vTaskDelay() 或等待信号量。
    • 挂起→就绪：调用 vTaskResume() 恢复任务。

  2. 调度策略

    类型 原理 适用场景
    抢占式调度 高优先级任务立即抢占CPU 工业控制（紧急响应）
    时间片轮转 同优先级任务轮流执行（默认1ms/次） 公平调度需求场景

  3. 优先级反转与解决方案

    • 问题场景：低优先级任务C占用资源 → 中优先级任务B运行 → 高优先级任务A等待资源被阻塞。
    • 解决方案：
      • 优先级继承：C临时提升至A的优先级（FreeRTOS互斥锁默认支持）。
      • 优先级天花板：资源预设最高优先级，占用者直接提升。

🔗 三、同步与通信机制

  1. 队列（Queue）

    • 作用：任务/中断间传递数据（FIFO或LIFO）。
    • API示例：
      QueueHandle_t xQueue = xQueueCreate(5, sizeof(int)); // 创建队列
xQueueSend(xQueue, &data, portMAX_DELAY);          // 发送数据
xQueueReceive(xQueue, &received, portMAX_DELAY);    // 接收数据

  2. 信号量 vs 互斥量

    类型 用途 特性
    二值信号量 任务同步（如中断通知任务） 值仅0/1，无优先级继承
    互斥量 资源保护（如共享内存访问） 支持优先级继承

  3. 任务通知（Task Notification）

    • 优势：比队列/信号量更快（减少内存拷贝），适用单任务同步。
    • 场景：高频率事件通知（如传感器数据到达）。

⚡ 四、中断与资源管理

  1. ISR设计规范

    • 黄金法则：执行时间 < 10μs，禁止调用 printf()、malloc()。
    • 安全通信：使用 xQueueSendFromISR() 代替普通队列API。

  2. 临界区保护

    taskENTER_CRITICAL(); // 关中断
// 操作共享资源（如全局变量）
taskEXIT_CRITICAL();  // 开中断

    注意：临界区代码不超过10条指令，避免破坏实时性。

  3. 内存管理

    • 动态分配风险：碎片化导致分配失败（工业系统慎用）。
    • 替代方案：静态内存池（xTaskCreateStatic()）。

🛠️ 五、开发实践与调试

  1. 栈溢出检测

    • 方法1：检测栈指针越界（快但漏检率高）。
    • 方法2：栈底填充 0xA5，检测是否被修改（慢但更可靠）。

  2. 低功耗设计

    • Tickless模式：空闲时停系统时钟，功耗降至μA级。
    • 实践场景：电池供电的物联网设备。

  3. 调试工具

    • Tracealyzer：可视化任务调度时序。
    • 栈使用检测：uxTaskGetStackHighWaterMark() 获取栈峰值。

💎 附：10道经典面试题（含答案）

  1. 问：FreeRTOS中如何避免优先级反转？
    答：使用互斥锁（Mutex）并启用优先级继承机制。

  2. 问：中断中能否调用 xQueueSend()？
    答：不能！必须用 xQueueSendFromISR()，否则引发数据竞争。

  3. 问：任务栈大小如何确定？
    答：静态分析（局部变量+调用深度） + 动态检测（栈溢出钩子函数）。

  4. 问：二值信号量与互斥量区别？
    答：互斥量支持优先级继承，二值信号量无此功能。

持续巩固建议：

  1. 动手实验：用STM32实现多任务温控系统（传感器任务→处理任务→显示任务）。
  2. 调试训练：开启栈溢出检测，故意缩小任务栈观察崩溃现象。
  3. 扩展学习：《Mastering FreeRTOS》第7章（同步机制深度解析）。
