1. 智能指针（C++11）

  • unique_ptr

    独占所有权（不可拷贝），移动语义传递

    auto ptr = std::make_unique<int>(42);  // 自动释放内存
auto ptr2 = std::move(ptr);           // 所有权转移

  • shared_ptr

    引用计数共享所有权，循环引用需用weak_ptr解决

    auto obj = std::make_shared<MyClass>();
std::weak_ptr<MyClass> observer = obj;  // 避免循环引用

2. 移动语义（C++11）

  • 右值引用（&&）：优化资源转移（如避免深拷贝）

    class Vector {
public:
    Vector(Vector&& other) noexcept   // 移动构造函数 声明移动操作不抛异常，便于编译器优化
        : data_(other.data_), size_(other.size_) {
        other.data_ = nullptr;        // 置空原指针防双重释放, 确保原对象析构时不会释放已转移的内存
    }
private:
    int* data_;
    size_t size_;
};

  • std::move

    显式转换左值为右值，触发移动操作

    Vector v1;
Vector v2 = std::move(v1);  // 调用移动构造

触发移动构造的典型场景

  • 返回临时对象：

    Vector createVector() {
    Vector tmp;
    return tmp;  // 编译器自动优化为移动构造（RVO 或移动语义）
}
Vector v = createVector();  // 移动构造被调用

  • 显式使用 std::move

    Vector v1;
Vector v2 = std::move(v1);  // v1 的资源被转移到 v2，v1 失效

重要注意事项

  • 移动后原对象状态：

    原对象（如other）应进入有效但未定义状态通常只允许析构或重新赋值）。

    Vector v1(100);
Vector v2 = std::move(v1);
// v1.data_ 已为 nullptr，不能再访问 v1[0]！

  • 与析构函数的配合：

    移动后的原对象析构时必须安全（即delete nullptr是安全的。

  • 自赋值检查：

    移动赋值运算符中需检查if (this != &other)，但移动构造无需（因构造时this尚未存在）。

3. Lambda表达式

  • 捕获列表：

    1. [&]捕获引用，捕获 Lambda 所在作用域内所有被使用的外部变量的引用，允许在 Lambda 内部直接修改外部变量。

      ​特点​：
      修改外部变量会影响原始值
      需警惕悬垂引用（Lambda 生命周期 > 外部变量时）

      int a = 1, b = 2;
auto lambda = [&] { 
    a = 10;  // 修改外部变量 a
    return a + b; 
};
lambda();  // 输出 12
std::cout << a;  // 输出 10（外部 a 被修改）

    2. [=]捕获值，捕获 Lambda 所在作用域内所有被使用的外部变量的副本（值拷贝）。

      特点​：
      外部变量修改不影响 Lambda 内部的副本
      默认 const，需 mutable 才能修改副本（不影响外部）

      int x = 5;
auto lambda = [=]() mutable {
    x++;  // 修改副本（需 mutable）
    return x;
};
lambda();  // 输出 6（副本值）
std::cout << x;  // 输出 5（外部 x 不变）

    3. [this]捕获当前对象, 捕获当前类的 this 指针，允许 Lambda 访问类的成员变量和函数。

      特点​：
      本质是捕获 this 指针，通过 this->member 访问成员
      成员变量的值是实时的（非定义时拷贝）
      需警惕悬垂 this（对象销毁后调用 Lambda）

      class Widget {
    int value = 42;
public:
    auto getLambda() {
        return [this] { 
            return value;  // 等价于 this->value
        };
    }
};

Widget w;
auto func = w.getLambda();
std::cout << func();  // 输出 42
w.value = 100;       // 修改成员
std::cout << func();  // 输出 100（实时访问）
      std::vector<int> vec = {1, 2, 3};
std::for_each(vec.begin(), vec.end(), [](int x) {
    std::cout << x * 2;  // 输出2 4 6
});

    4. 空捕获列表 [] :
      ​不捕获任何局部变量

      void example() {
    int local = 10;
    auto lambda = [] { 
        // 错误！无法访问 local
        // std::cout << local; 
    };
}

      可访问全局变量和静态变量

      int global = 20;
void example() {
    static int s = 30;
    auto lambda = [] {
        std::cout << global; // ✅ 合法（全局变量）
        std::cout << s;      // ✅ 合法（静态变量）
    };
}

      不能直接访问成员变量
      ​参数和函数体不受影响

      auto add = [](int a, int b) { 
    return a + b; // ✅ 仅依赖参数
};

    5. 混合捕获：

      int a = 1, b = 2;
auto lambda = [=, &b] {  // 按值捕获 a，按引用捕获 b
    return a + b;  
};
b = 3;
lambda();  // 输出 4（a=1 快照 + b=3 实时）

4. 类型推导与结构化绑定（C++17）

  • auto：自动推导变量类型

    auto dict = std::map<std::string, int>{{"A", 1}, {"B", 2}};

  • 结构化绑定：解包复合类型

    for (const auto& [key, value] : dict) {  // 直接访问键值对
    std::cout << key << ":" << value;
}

5. 编译时计算

  • constexpr函数：编译期执行计算

    constexpr int factorial(int n) {
    return (n <= 1) ? 1 : n * factorial(n - 1);
}
static_assert(factorial(5) == 120);  // 编译期断言

6. 线程管理

  • std::thread：创建线程

    void task() { /* 耗时操作 */ }
std::thread t(task);  // 启动线程
t.join();             // 等待结束

  • 异步任务：

    std::async + std::future

    auto future = std::async(std::launch::async, []{ return 42; });
int result = future.get();  // 阻塞获取结果
    • std::async​：启动异步任务，返回std::future对象
      • std::launch::async：强制在新线程中执行（适合CPU密集型任务）
      • std::launch::deferred：延迟执行（调用get()时在当前线程执行，适合资源敏感型任务）
      • 默认策略（async|deferred）：由编译器决定执行方式（需谨慎使用）
    • std::future​：异步结果的“提货单”，提供阻塞或非阻塞的结果获取方式
      • 任务返回值存入共享状态（shared state），std::future通过指针关联此状态
      • future.get()：​阻塞当前线程
        • 替代方案：用wait_for()实现超时等待
        if (future.wait_for(1s) == std::future_status::ready) {
    result = future.get(); // 非阻塞获取结果
} else {
    // 处理超时或任务未完成
}
      • 异步任务中的异常会通过get()重新抛出到调用线程： 
        try {
    future.get();
} catch (const std::exception& e) {
    std::cerr << "异步任务异常: " << e.what();
}
      • std::future不可复制，但支持移动语义（future2 = std::move(future1)）
  1. 同步机制

  • 互斥锁：std::mutex + std::lock_guard

    std::mutex mtx;
void safe_push(std::vector<int>& vec, int val) {
    std::lock_guard<std::mutex> lock(mtx);  // 自动加锁/解锁
    vec.push_back(val);
}

  • 原子操作：std::atomic无锁并发

    std::atomic<int> counter(0);
counter.fetch_add(1, std::memory_order_relaxed);  // 无锁递增

    • 原子操作的核心作用​

      1. ​解决数据竞争​：
        普通变量（如int counter）在多线程中同时读写时，可能因操作被中断导致结果错误（例如两个线程同时读取counter=0，各自加1后写回，结果变为1而非2）。
      2. ​原子性保证​：
        std::atomic确保操作（如fetch_add）是不可分割的。例如，当一个线程执行fetch_add(1)时，其他线程无法同时修改counter，从而避免数据竞争

    • 无锁并发的实现原理

      1. ​硬件级原子指令​：
        std::atomic底层通过CPU指令（如x86的LOCK CMPXCHG）实现原子操作，无需软件锁（如std::mutex）。
      2. ​性能优势​：
        相比互斥锁（std::mutex）的上下文切换开销，原子操作直接操作内存，速度更快，尤其适合高频简单操作（如计数器）
      // 传统锁方案 vs 原子操作方案
std::mutex mtx;
int counter = 0;

// 传统锁：每次操作需加锁/解锁
mtx.lock();
counter++;  
mtx.unlock();

// 原子操作：无锁直接操作
atomic_counter.fetch_add(1);

    • 何时使用原子操作（vs 互斥锁）

      场景 推荐方案 原因
      简单变量（int、bool等） std::atomic 无锁高性能，避免锁开销
      复杂操作（如链表更新） std::mutex 原子操作难以保证多个步骤的原子性（需CAS循环，实现复杂）
      读写混合且高频 读写锁（std::shared_mutex） 减少写阻塞，提升并发读性能

    • 原子操作的其他常见方法

      1. ​加载/存储​：
        counter.load() 获取当前值，counter.store(42) 设置新值。
      2. ​比较与交换（CAS）​​：
      int expected = counter.load();
while(!counter.compare_exchange_weak(expected, new_value)) {}

      用于无锁数据结构（如队列），需循环直到更新成功

​高频面试考点解析​

  • ​Q：智能指针如何避免内存泄漏？​​
    ​A​：unique_ptr通过独占所有权防拷贝；shared_ptr用引用计数自动释放；循环引用场景需搭配weak_ptr。
  • ​Q：移动构造函数与拷贝构造的区别？​​
    ​A​：移动构造“窃取”资源（原对象置空），避免深拷贝开销；拷贝构造创建独立副本。
  • ​Q：C++17结构化绑定的应用场景？​​
    ​A​：简化std::pair/tuple/结构体的访问，提升代码可读性（如遍历map）
