每日知识点-4

1. 虚函数表(vtable)的实现原理及多态性能损耗

  • 原理：每个包含虚函数的类会生成一个虚函数表，存储虚函数指针；对象内存首部包含指向该表的指针(vptr)。动态绑定通过运行时查找虚函数表实现。
  • 性能损耗：多态调用需额外查表跳转(约 1-2 个时钟周期)，嵌入式场景中需权衡灵活性与实时性。
  • 项目关联：在 OpenCV 图像处理基类中，用虚函数实现不同算法的多态调用(如 FeatureDetector::detect())。

2. 智能指针的线程安全性及 std::shared_ptr 循环引用问题

  • 线程安全：std::shared_ptr 的引用计数原子操作保证线程安全，但指向的堆对象需额外同步。
  • 循环引用：若 A 持有 B 的shared_ptr, B 也持有 A 的 shared_ptr, 引用计数永不归零(两个气球相互牵引，彼此都无法被释放)。
  • 解决方案：用 std::weak_ptr 替代单向持有关系 (如观察者模式中的订阅者列表)。
  • 项目关联：在模型部署框架中管理 TensorRT 引擎时，用 unique_ptr 避免循环引用导致内存泄漏。

3. const 成员函数与 mutable 关键字的应用场景

  • const 成员函数：承诺不修改对象状态 (如 size() const)，允许 const 对象调用。
  • mutable: 修饰可能被 const 函数修改的成员 (如缓存标记、 互斥锁)。
  • 项目关联：在实时数据采集类中，用 mutable std::mutex 保证 const 函数内的线程安全。

4. C++11 右值引用与移动语义的优化原理。

  • 右值引用(&&)：标识临时对象，允许资源转移而非拷贝(如 std::move())。
    右值引用可以直接指向右值，也可以通过std::move指向左值；而左值引用只能指向左值(const左值引用也能指向右值)。
    作为函数形参时，右值引用更灵活。虽然const左值引用也可以做到左右值都接受，但它无法修改，有一定局限性。
  • 移动构造函数: 接管原对象资源(如指针)，将原对象置空避免双重释放，避免了传统的深拷贝的消耗，但是原对象被释放。

5. 嵌入式场景中 volatile 与内存屏障的作用

  • volatile：阻止编译器优化对变量的读写(如硬件寄存器映射变量)。
  • 内存屏障：确保指令执行顺序(如 __sync_synchronize() 防止多核缓存不一致)。
  • 项目关联：在 STM32 中断服务程序中，用 volatile 修饰全局计时器变量，配合内存屏障保证数据可见性。

6. 进程间通信 (IPC) 的5种方式及适用场景

  • 管道 (Pipe) : 单向通信，父子进程间使用 (如 Shell 命令的 | 操作)。
  • 共享内存：高效大数据传输，需同步机制 (如信号量)。
  • 消息队列：结构化数据传递，支持多线程异步通信。
  • 信号 (Signal) : 简单事件通知 (如 SIGINT 终止进程)。
  • 套接字 (Socket) : 跨网络或本机进程通信。

7. 虚拟内存与物理内存的映射机制 (MMU 作用)

  • MMU 功能：
    • 虚拟地址 -> 物理地址转换 (页表查询)。
    • 内存权限检查 (读/写/执行)。
    • TLB 缓存加速地址转换。
  • 项目关联：在嵌入式 AI 推理中，MMU 隔离模型权重内存区，防止非法访问。

8. Linux 内核中断处理流程 (上半部 vs 下半部)

  • 上半部 (Top Half) : 在中断上下文中快速响应 (如置标志位)，不可阻塞。
  • 下半部 (Bottom Half) : 延后处理耗时操作 (如 tasklet 或 workqueue), 可调度。
  • 项目关联：摄像头数据采集时，上半部读取硬件缓冲区，下半部进行图像预处理。

9. 用户态与内核态切换的性能损耗及优化方法。

  • 损耗来源：上下文保存/恢复、CPU模式切换(约数百纳秒)。
  • 优化方法：
    • 减少系统调用次数(如批量读写)
    • 使用mmap映射文件到用户空间。
    • 内核旁路技术(如 DPDK)

10. 编写 Linux 内核模块的基本步骤

  1. 实现模块初始化函数 (module_init()) 和退出函数 (module_exit())
  2. 注册设备驱动或文件系统接口
  3. 处理并发 (如自旋锁 spin_lock())
  4. 编译为 .ko 文件并加载 (insmod)
  • 项目关联：在 Jetson Nano 上开发自定义 GPIO 驱动时，需编写内核模块实现引脚控制。

如何设计线程安全的环形缓冲区(Ring Buffer)？

  • 代码如下： 
    class RingBuffer {
public:
    RingBuffer(size_t size) : buf_(size), head_(0), tail_(0) {}
    bool push(unit8_t data) {
        std::lock_guard<std::mutex> lock(mutex_);
        if ((head_ + 1) % buf_.size() == tail_) return false;   // 满 
        buf_[head_] = data;
        head_ = (head_ + 1) % buf_size();
        return true;
    }

    bool pop(unit8_t& data) {
        std::lock_guard<std::mutex> lock(mutex_);
        if (head_ == tail_) return false;   // 空
        data = buf_[tail_];
        tail_ = (tail_ + 1) % buf_.size();
        return true;
    }

private:
    std::vector<unit8_t> buf_;
    size_t head_, tail_;
    std::mutex mutex_;
};
