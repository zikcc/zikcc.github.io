1. 指针与引用的核心区别？项目中如何应用？

  • 区别：
    • 指针是变量存储地址，可重指向其他对象；引用是别名，绑定后不可更改。
    • 指针可为空(nullptr), 引用必须绑定有效对象。
  • 项目关联：
    在OpenCV图像处理中，使用指针传递大型图像数据矩阵(如cv::Mat*)避免拷贝；引用用于函数参数传递(如 const cv::Mat&)保证安全访问。
    例如:
    • 使用指针: 
      void proccessImage(cv::Mat* img) {
    // 假设对图像进行灰度化处理
    cv::cvColor(*img, *img, cv::COLOR_BGR2GRAY);
}
      不加const修饰时，可改变指针指向，和指向的值。
      常量指针(const cv::Mat* img)：不能通过该指针修改指针指向的值。
      指针常量(cv::Mat* const img)：不能修改指针的指向(也就是指针指向的地址不能变)
    • 使用引用: 
      void proccessImage(const cv::Mat& img) {
    // 灰度化处理
    cv::Mat grayImg;
    cv::cvtColor(img, grayImg, cv::COLOR_BGR2GRAY);

}
    最佳实践:
    • 如果只是函数参数传递，优先使用 const cv::Mat&（更安全、更简洁）
    • 如果需要处理 nullptr (例如void proccessImage(const cv::Mat* img = nullptr) {} ) 或动态切换对象，使用 const cv::Mat*。

    常量指针(const cv::Mat* img)是否一定能保证指向的值不会被改变？
    不一定！const cv::Mat* img（指向常量的指针）只能保证不能通过该指针修改数据，但原始数据仍可能被其他方式修改。

2. const 关键字在函数参数和成员函数中的作用？

  • 参数: const T& 防止修改输入数据(如传感器的原始数据)。
  • 成员函数: void func() const 承诺不修改对象状态(如线程安全的配置类), 表示只读。
    例如: 
    class Config {
private:
    int width_;
    int height_;
    mutable int accessCount_;   // 可被 const 函数修改
public:
    // const 成员函数，承诺不修改对象状态
    void printConfig() const {
        // width_ = 100;    // 错误，不能修改非 mutable 成员
        accessCount_++;     // 正确，mutable成员可修改
        std::cout << "Width: " << width_ << ", Height: " << height_ << std::endl;
    }
}
    返回非 const 指针/引用是危险的 
    class BadExample {
private:
    std::vector<int> data_;
public:
    // 危险：const 函数暴露了可修改的内部数据
    std::vector<int>& getData() const { return data_; }  // 不要这样做！
};
    const 成员函数和非 const 成员函数可以重载
  • 项目关联: 在模型推理的类中，const修饰推理函数确保权重不被意外修改。

3. 手写单例模式(线程安全版)

class Singleton {
public:
    static Singleton& getInstance() {
        static Singleton instance;  // 即使多次访问这个函数，instance也只会初始化一次。
        return instance;
    }
    Singleton(const Singleton&) = delete;   // 禁止拷贝构造
    Singleton& operator=(const Singleton&) = delete;    // 禁用赋值构造
private:
    Singleron() = default;  // 私有构造函数
}
  • 项目关联: 用于全局配置管理(如摄像头参数加载)，避免多线程重复初始化。

4. ps -aux 和 top 命令的区别？项目中如何用？

  • ps -aux: 静态快照显示所有进程信息(如PID，CPU占用)
  • top: 动态实时监控进程资源(按CPU排序，支持交互命令)
  • 项目关联: 部署YOLO模型时用top监控推理进程的CPU/内存峰值，优化线程数。

5. 如何用grep 搜索日志中的"segmentation fault"?

grep "segmentation fault" /var/log/syslog | tail -n 20  # 显示最后20行
  • 项目关联: 在嵌入式设备调试中，快速定位模型推理崩溃的堆栈信息。

6. 解释 strace 命令的作用并举例

  • 作用: 追踪进程的系统调用(如文件读写、信号处理)
  • 示例: 
    strace -o debug.log ./inference_app  # 记录系统调用到文件
  • 项目关联: 分析 OpenCV 视频采集卡顿问题，发现 read() 调用阻塞事件过长。

7. 链表反转(手写代码 + 优化思路)

ListNode* reverseList(ListNode* head) {
    ListNode* prev = nullptr, *curr = head;
    while (curr) {
        ListNode* next = curr -> next;
        curr -> next = prev;
        prev = curr;
        curr = next;
    }
    return prev;
}
  • 优化思路: 嵌入式设备中避免递归防止栈溢出，用迭代法减少内存消耗。

8. 哈希表冲突解决方法的优缺点?

  • 开放寻址: 内存紧凑但易聚集(适合嵌入式小内存场景)
  • 链地址法: 稳定但须额外指针(PC端常用)
  • 项目关联: 在特征点匹配算法中，用链地址法哈希表加速特征查询。

9. 快速排序实现及时间复杂度分析

// 分区函数：选择 pivot，将小于 pivot 的元素放在左边，大于 pivot 的放在右边
int partition(std::vector<int>& arr, int low, int high) {
    int pivot = arr[high];   // 选择最后一个元素作为 pivot
    int i = low - 1;     // i 是小于 pivot 的区域的边界
    for (int j =  low; j < high; ++j) {
        if(arr[j] < pivot) {
            ++i;
            std::swap(arr[i], arr[j]);
        }
    }
    std::swap(arr[i + 1], privat);   // 将 pivot 放到正确的位置
    return i + 1;     // 返回 pivot 的索引
}
void quickSort(std::vector<int>& arr, int low, int high) {
    if (low < high) {
        int pivotIndex = partition(arr, low, high);
        quickSort(arr, low, pivotIndex - 1);
        quickSort(arr, pivotIndex + 1, high);
    }
}
// 递归版本可能在大数据量时导致 栈溢出, 可以用栈模拟递归：
void quickSortIterative(std::vector<int>& arr, int low, int high) {
    std::stack<std::pair<int, int>> st;
    st.push({low, high});
    while (!st.empty()) {
        auto {l, h} = st.top();
        st.pop();
        if (l >= h) continue;
        int pivotIndex = partition(arr, l, h);
        st.push({l, pivotIndex - 1});
        st.push({pivotIndex + 1, h});
    }
}

10. 用栈实现队列的功能

class MyQueue {
    stack<int> in, out;
public:
    void push(int x) {
        in.push(x);
    }
    int pop() {
        if (out.empty()) {
            // 如果 out 为空，那就把 in 的所有元素倒过来, 也就是压入 out中
            while (!in.empty()) {
                out.push(in.top());
                in.pop();
            }
        }
        int val = out.top();
        out.pop();
        return val;
    }
    int peek() {
        if (outStack.empty()) {
            // 如果 outStack 为空，把 inStack 的所有元素倒过来
            while (!inStack.empty()) {
                outStack.push(inStack.top());
                inStack.pop();
            }
        }
        return outStack.top();  // 只是观察，不删除元素
    }
}
