GDB_调试指南

zikcc2025-05-242025-07-09

以下是一份详细的 GDB 调试指南，涵盖基础到高级用法，适用于 Linux C/C++ 程序调试（包括多线程、核心转储分析等场景）。

1. GDB 基础使用

1.1 启动 GDB

# 调试可执行文件
gdb ./your_program

# 调试正在运行的进程
gdb -p <PID>

# 调试核心转储文件（coredump）
gdb ./your_program core

1.2 常用基础命令

命令 缩写 作用
run r 启动程序
break <location> b 设置断点（如 b main, b file.cpp:10）
continue c 继续执行到下一个断点
next n 单步执行（不进入函数）
step s 单步执行（进入函数）
print <var> p 打印变量值
backtrace bt 查看调用栈
quit q 退出 GDB

2. 断点管理

2.1 设置断点

b main          # 在 main 函数开头断点
b file.cpp:20   # 在 file.cpp 的第 20 行断点
b *0x4005a6     # 在内存地址断点（反汇编调试）

2.2 条件断点

b foo if x > 10   # 当 x > 10 时在 foo 函数中断

2.3 查看/删除断点

info breakpoints  # 查看所有断点
delete <num>      # 删除指定编号的断点
clear             # 清除当前行的断点

3. 变量与内存操作

3.1 查看变量

p x              # 打印变量 x
p/x x            # 以十六进制打印
p *ptr@10        # 打印指针 ptr 指向的 10 个元素

3.2 修改变量

set var x = 5    # 修改变量 x 的值

3.3 检查内存

x/8wx 0x400000   # 查看内存（8 个 4 字节字，十六进制）
x/s ptr          # 打印字符串（直到 '\0'）

4. 多线程调试

4.1 线程控制

info threads     # 查看所有线程
thread <ID>      # 切换到指定线程
b foo thread 2   # 仅在线程 2 的 foo 函数中断

4.2 锁定线程调度

set scheduler-locking on  # 仅当前线程运行（其他暂停）
set scheduler-locking off # 恢复线程自由切换

4.3 死锁调试

catch syscall pthread_mutex_lock  # 捕获锁调用
thread apply all bt              # 查看所有线程堆栈

5. 核心转储（Core Dump）分析

5.1 生成 Core Dump

ulimit -c unlimited  # 允许生成 core 文件
./your_program       # 崩溃后生成 core

5.2 分析 Core

gdb ./your_program core
bt                   # 查看崩溃时的调用栈
p <var>              # 检查变量状态

6. 高级功能

6.1 反汇编调试

disassemble          # 查看当前函数的汇编代码
si / ni              # 汇编级单步执行（step/next instruction）

6.2 自定义命令

在 ~/.gdbinit 中定义快捷命令：

define print_array
  set $i = 0
  while $i < $arg0
    printf "arr[%d] = %d\n", $i, arr[$i]
    set $i = $i + 1
  end
end

使用时：print_array 10。

6.3 反向调试（Reverse Debugging）

record full          # 开始记录执行流程
reverse-step         # 反向执行

7. 调试优化后的代码

如果程序编译时用了 -O2，变量可能被优化掉，需：

# 禁用优化或添加调试符号
gcc -O0 -g program.c

8. 实用技巧

  • 日志输出：
    printf "x=%d\n", x（在 GDB 中直接打印日志）。
  • 跳过代码块：
    until <line>（执行到指定行）。
  • 调试子进程：
    set follow-fork-mode child（跟踪子进程）。

9. 推荐学习资源

  • 官方手册：gdb -tui（文本界面模式）或 info gdb。
  • 图形化前端：
    • cgdb（增强型命令行 GDB）
    • ddd（图形化调试器）。

掌握这些命令后，你可以高效诊断段错误、死锁、数据竞争等问题。对于复杂项目，建议结合 VS Code + GDB 插件 提升效率。
