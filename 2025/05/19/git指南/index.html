git指南

新建仓库

远程仓库

在 github 上创建新仓库

本地仓库

github 已经创建新仓库，使用以下命令初始化本地仓库并推送到远程仓库

# 进入项目根目录
cd ~/your-project-folder

# 初始化 Git 仓库
git init

# 创建 .gitignore 文件（可选）
echo "*.log" >> .gitignore
echo "tmp/" >> .gitignore

# 添加所有文件到暂存区
git add .

# 提交代码
git commit -m "初始提交：***项目"

# 关联远程仓库（替换 YOUR-USERNAME）
git remote add origin https://github.com/YOUR-USERNAME/nginx-streaming-demos.git

# 推送到 GitHub
git branch -M main
git push -u origin main

已有远程仓库与本地仓库

有修改，需更新

# 查看当前分支
git branch
# 如果要切换到其他分支
git checkout <branch-name>

# 将更改添加到本地暂存区
git add .
# 如果只是指定文件
git add <file-name>

# 提交更改
git commit -m "Your commit message"

# 推送
git push
# 如果没有设置上游分支，要指定分支
git push origin <branch-name>

完整重置仓库

# 备份当前仓库
cd ..
cp -r current_project project-bak

# 重新克隆仓库
git clone <您的远程仓库地址> project-new

# 验证新仓库
cd project-new
git branch -a  # 应正常显示

一、Git 基础概念

  1. 仓库（Repository）

    • 存储代码历史与版本的核心目录，通过

      初始化本地仓库。

    • 克隆远程仓库：

      git clone <仓库地址>

      （自动关联远程源）。

  2. 工作流程三区

    • 工作区：本地修改文件的目录。

    • 暂存区（Staging Area）：

      临时保存待提交的修改（

      git add

      添加至此）。

    • 本地仓库：

      通过

      git commit

      将暂存区内容永久存储为版本。

二、环境配置

  1. 安装 Git

    • Windows：

      官网下载安装包。

    • macOS：

      brew install git

      （Homebrew）或 Xcode Command Line Tools。

    • Linux：

      sudo apt install git

      （Debian/Ubuntu）或

      sudo yum install git

      （CentOS）。

  2. 全局配置

    git config --global user.name "Your Name"
git config --global user.email "your@email.com"
git config --global color.ui true  # 启用颜色高亮

三、日常操作命令

操作 命令示例 说明
查看状态 git status 显示工作区/暂存区状态
添加文件到暂存区 git add <文件名> 或 git add . 跟踪修改/新增文件
提交到本地仓库 git commit -m "描述" 生成新版本记录
查看提交历史 git log --oneline --graph 简洁化日志（含分支拓扑）
推送代码到远程 git push origin <分支名> 首次推送加 -u 设置上游
拉取远程更新 git pull --rebase 变基拉取（保持历史线性）
撤销工作区修改 git checkout -- <文件名> 放弃未暂存的修改
撤销暂存区文件 git reset HEAD <文件名> 移出暂存区

四、分支管理

  1. 分支操作

    • 创建并切换分支：

      git checkout -b feature/new

    • 合并分支：先切回主分支

      git checkout main

      ，再执行

      git merge feature/new

    • 删除分支：

      git branch -d feature/new

      （安全删除）。

  2. 分支策略（推荐）

    分支类型 用途
    main/master 稳定生产版本，禁止直接修改3****10。
    develop 集成开发分支，测试稳定后合并至主分支3****10。
    feature/* 新功能开发分支3****10。
    hotfix/* 紧急生产环境修复3****10。

五、高级技巧与最佳实践

  1. 代码回退与恢复

    • 回退到上一版本：

      git reset --hard HEAD^

    • 恢复误删提交：

      git reflog

      查历史操作

      git reset --hard <commit-id>

  2. 冲突解决

    • 拉取时冲突：手动修改冲突文件

      git add <文件>
      git commit

      完成合并。

    • 图形化工具：

      git mergetool

      （需配置 diff 工具如 VSCode）。

  3. 忽略文件（.gitignore）

    • 创建文件，排除日志、编译产物等：

      node_modules/
.env
*.log
dist/

    • 生效步骤：若已跟踪文件需先删除缓存

      git rm -r --cached .

      重新提交。

  4. 提交规范
    推荐语义化提交（Conventional Commits）：

    git commit -m "feat: 新增用户登录功能"
git commit -m "fix: 修复支付接口崩溃 #123"

    类型：feat（功能）、fix（修复）、docs（文档）、refactor（重构）等。

六、学习资源与工具

  • 交互式练习：

    Learn Git Branching（可视化分支操作）。

  • 图形化客户端：

    • GitHub Desktop（新手友好）
    • VS Code + GitLens 插件（代码追溯增强）

  • 官方文档：

    Git Book（全面详解）

掌握 Git 的核心在于 理解工作流三区 和 分支管理逻辑，配合规范提交与团队协作策略，可大幅提升开发效率。建议从基础命令入手，逐步实践分支合并与冲突解决，再深入高级功能（如 rebase、stash）。
